# 企业漂绿测算系统 — 技术文档

> **基于异构大语言模型集成推理的绿色信息披露质量评估系统**
>
> 参考论文：*How Environmental Courts Inhibit Corporate Greenwashing—Evidence from Heterogeneous LLM Measures*

---

## 摘 要

本系统针对企业环境信息披露中的"漂绿"现象，构建了一套基于异构大语言模型（Heterogeneous LLM）集成推理的自动化测算系统。系统采用三模型（DeepSeek-R1-32B、Qwen-3-32B、GLM-4.7）独立判别 + 多数投票确权 + 集成平均评分的技术路线，实现对企业年报中环境披露内容的深度语义分析。

系统提供可视化Web交互界面，支持PDF/DOCX/TXT多格式年报上传、单企业分析、批量对比、行业基准分析等功能，界面采用森林绿主题设计，符合WCAG AA级无障碍标准。

## 一、项目概述

### 1.1 研究背景

随着ESG投资理念的兴起，企业环境信息披露质量成为资本市场关注的焦点。然而，部分企业存在"漂绿"（Greenwashing）行为——即通过夸大环保成效、使用模糊修辞等方式，营造积极环保形象但实际行动不足。传统的人工审核方式效率低、主观性强，难以满足大规模监管需求。

### 1.2 核心问题

如何利用人工智能技术，**自动化、规模化、客观地**识别和度量企业环境披露中的漂绿程度？

### 1.3 解决方案

本系统借鉴论文提出的异构大语言模型集成推理框架，通过以下核心思路解决问题：

1. **语义维度**：区分"实质性陈述"（有具体数据支撑）与"描述性陈述"（口号式修辞）
2. **情感维度**：量化描述性陈述的"环境语调"（Environmental Tone）积极程度
3. **基准对比**：以行业中位数为基准，计算企业相对漂绿程度
4. **多模型融合**：三个异构大模型独立判别，通过多数投票和集成平均降低偏差

### 1.4 系统功能

| 功能模块 | 功能描述 |
|----------|----------|
| 文件上传 | 支持 PDF、DOCX、TXT 三种格式的年报文件上传，自动提取文本 |
| 单企业分析 | 输入一家企业的年报，输出漂绿指数、分类统计、语句详情 |
| 批量分析 | 同时上传多家企业年报，批量测算并支持横向对比 |
| 行业基准 | 基于同行业企业的漂绿指数分布，计算行业基准与排名 |
| 结果可视化 | 仪表盘、环形图、柱状图、折线图等多种图表展示 |
| 演示模式 | 内置Mock LLM，无需API Key即可体验完整功能 |

### 2.2 解题思路详解

#### 思路一：从"陈述性质"切入

企业漂绿的核心特征是**"说得多、做得少"**。因此，我们将环境相关语句分为两类：
- **实质性陈述**（Substantive）：包含可验证的定量数据或具体成果（如"减排15%"、"投入5000万"、"通过ISO14001认证"）
- **描述性陈述**（Descriptive）：定性的口号或模糊承诺（如"积极推动绿色发展"、"高度重视环保"）

**漂绿的信号**：描述性陈述占比过高 + 语调过于积极。

#### 思路二：从"相对水平"度量

漂绿是一个**相对概念**——同样的语调，在重污染行业可能是正常的，但在轻污染行业可能就偏高。因此：
- 以**同行业、同年度**的企业环境语调中位数为基准
- 企业语调高于行业中位数越多，漂绿嫌疑越大

#### 思路三：用多模型对抗偏差

单个大模型可能存在系统性偏差（如某模型倾向于把句子都判为正面）。使用三个**异构架构**的模型：
- 不同的训练数据 → 不同的知识背景
- 不同的模型结构 → 不同的推理方式
- 通过投票和平均 → 抵消个体偏差，提升鲁棒性

## 四、模型架构与关键技术

### 4.1 异构大模型选型

参照论文配置，系统集成三个异构架构的大语言模型，形成"专家评审团"：

| 模型名称 | 架构类型 | 参数规模 | 定位 | 特点 |
|----------|----------|----------|------|------|
| **DeepSeek-R1-32B** | Decoder-Only | 32B | 推理专家 | 推理能力强，逻辑严谨，适合分类任务的精确判断 |
| **Qwen-3-32B** | Decoder-Only | 32B | 中文专家 | 中文理解优秀，综合能力强，适合语义细腻的情感分析 |
| **GLM-4.7** | Decoder-Only | 4.7B | 通用对话模型 | 中文理解能力强，支持多轮对话与复杂任务，适合文本分类与情感分析 |

**异构性价值**：
- 架构差异（均为 Decoder-Only 架构，但训练数据和优化目标不同）→ 不同的推理模式
- 训练数据差异 → 不同的知识覆盖面
- 规模差异 → 不同的判断 granularity

### 4.2 提示词工程（Prompt Engineering）

#### 4.2.1 分类任务提示词

采用**零样本思维链（Zero-shot Chain-of-Thought）** 范式：

```
角色：你是一位企业环境信息披露质量评估专家。

任务：判断下面这句话属于以下哪一类：
  A. 实质性陈述 — 包含可验证的定量数据或具体环保成果
  B. 描述性陈述 — 定性的口号或模糊的环保承诺，无具体数据支撑
  C. 非环保语句 — 虽然可能包含环保关键词，但语境与环境治理无关

语句：{sentence}

请先给出你的分析推理过程（不要超过50字），
然后给出你的最终判断（只输出字母A/B/C）。

你的回答格式：
推理：<你的推理过程>
答案：<A/B/C>
```

#### 4.2.2 情感分析提示词

采用**语境感知 + 连续评分**范式：

```
角色：你是一位企业环境语调分析专家。

任务：请从-1到1给下面这句话的环境语调打分：
  -1 = 非常负面（承认环境问题、投入不足、表现差等）
   0 = 中性（客观陈述、不褒不贬）
  +1 = 非常正面（强调环保成效、积极承诺、成绩显著等）

语句：{sentence}

请先分析这句话的语境和情感倾向，然后给出评分。

你的回答格式：
分析：<你的分析>
评分：<从-1.0到1.0之间的数字，保留两位小数>
```

### 4.3 模型融合策略

#### 4.3.1 分类任务：多数投票原则（Majority Voting Rule）

```
三个模型独立判断 → 统计投票结果 → 输出最终类别 + 置信度
```

**投票结果类型**：

| 类型 | 票型 | 置信度 | 处理方式 | 占比（论文数据） |
|------|------|--------|----------|-----------------|
| 全票通过 | 3:0 | 1.00 | 直接采纳 | ~89.37% |
| 多数通过 | 2:1 | 0.67 | 采纳多数，抽样复核 | ~9.81% |
| 完全分歧 | 1:1:1 | 0.33 | 标记高歧义，人工复核 | ~0.82% |

**Fleiss' Kappa 一致性检验**：

$$\kappa = \frac{P_o - P_e}{1 - P_e}$$

- $P_o$：观测一致率（三个模型意见一致的比例）
- $P_e$：机遇一致率（随机猜测可能的一致比例）

**论文报告值**：Fleiss' Kappa = 0.84 → 模型间**高度一致**

#### 4.3.2 评分任务：集成平均法（Ensemble Averaging）

$$s_{final} = \frac{1}{M} \sum_{m=1}^{M} s_m$$

- $M$：模型数量（M=3）
- $s_m$：第 m 个模型的评分

支持多种平均策略：
- **算术平均**（默认）：简单直接，无先验假设
- **加权平均**：根据模型表现分配权重
- **中位数**：对异常值更鲁棒
- **截尾平均**：去掉最高最低各 25%，取中间 50% 的平均

### 4.4 漂绿指数计算方法

#### 4.4.1 平均环境语调（Average Environmental Tone）

$$Tone_{i,t} = \frac{1}{N_{i,t}} \sum_{j=1}^{N_{i,t}} s_{i,t,j}$$

- $Tone_{i,t}$：企业 i 在 t 年的平均环境语调
- $N_{i,t}$：企业 i 在 t 年的描述性环境语句总数
- $s_{i,t,j}$：第 j 条描述性语句的情感得分（[-1, 1]）

> **为什么只用描述性语句算语调？**
> 因为实质性语句有客观数据支撑，其语调是"事实"而非"修辞"。漂绿主要通过描述性语句的过度积极修辞来实现。

#### 4.4.2 漂绿指数（Greenwash Index）

$$Greenwash_{i,t} = Tone_{i,t} - Median(Tone_{industry,t})$$

- $Greenwash_{i,t}$：企业 i 在 t 年的漂绿指数
- $Median(Tone_{industry,t})$：t 年同行业所有企业环境语调的中位数

**指数解读**：

| 漂绿指数范围 | 等级 | 含义 |
|-------------|------|------|
| < -0.1 | 保守披露 | 环境语调低于行业水平，披露偏保守 |
| -0.1 ~ 0.1 | 正常披露 | 环境语调与行业持平，属于正常披露 |
| 0.1 ~ 0.3 | 轻微漂绿 | 语调略高于行业，存在轻度漂绿嫌疑 |
| 0.3 ~ 0.6 | 中度漂绿 | 语调明显高于行业，漂绿嫌疑较大 |
| > 0.6 | 严重漂绿 | 语调远高于行业，高度疑似漂绿 |

### 4.5 数据处理关键技术

#### 4.5.1 环境关键词体系（五大维度）

系统构建了覆盖五大维度的扩展关键词向量库：

| 维度 | 关键词示例 |
|------|-----------|
| 污染防治 | 污染治理、减排、脱硫脱硝、污水处理、废气治理、固废处置 |
| 资源利用 | 节能、清洁能源、可再生能源、资源回收、循环利用、节水 |
| 绿色认证 | ISO14001、绿色工厂、环保认证、碳中和认证、清洁生产 |
| 碳排放管理 | 碳排放、碳中和、碳达峰、碳交易、CCUS、碳足迹 |
| ESG与可持续发展 | ESG、可持续发展、绿色金融、社会责任、双碳、美丽中国 |

#### 4.5.2 文本清洗四步法

```
原始文本
  ↓
① 规范化 → 去除空白、全角转半角、清理不可见字符
  ↓
② 句子切分 → 按标点（。！？；）切分 + 合并过短片段
  ↓
③ 去重处理 → MD5哈希精确去重 + 相似句检测
  ↓
④ 质量过滤 → 长度过滤(8-500字) + 噪音模式匹配(标题/页码/表头)
  ↓
清洗后的语句列表
```

#### 4.5.3 多引擎 PDF 解析 Fallback

为确保不同PDF文件都能成功解析，系统采用四级 fallback 机制：

```
pdfplumber（首选，表格支持好）
    ↓ 失败
PyPDF2（轻量快速）
    ↓ 失败
pypdf（PyPDF2继任者）
    ↓ 失败
pdfminer.six（兼容性最强）
    ↓ 失败
抛出明确错误，提示用户检查文件
```

### 5.2 对照实验设计

#### 实验一：多模型 vs 单模型 准确率对比

**假设**：多模型集成的准确率和稳定性优于任何单个模型。

| 方案 | 模型 | 预期准确率 | 预期一致性 |
|------|------|-----------|-----------|
| 单模型A | DeepSeek-R1-32B | ~82% | - |
| 单模型B | Qwen-3-32B | ~80% | - |
| 单模型C | GLM-4.7 | ~85% | - |
| 三模型集成 | 多数投票 | **~91%** | Fleiss' Kappa = 0.84 |

**验证方法**：在人工标注的测试集上分别计算各方案的准确率、精确率、召回率、F1值。

#### 实验二：不同融合策略对比

| 策略 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 多数投票 | 分类任务 | 简单直观、易解释 | 未考虑置信度差异 |
| 加权投票 | 有模型先验表现 | 可利用模型表现差异 | 权重需要验证 |
| 集成平均 | 评分任务 | 平滑结果、降低方差 | 可能被极端值影响 |
| 中位数聚合 | 评分任务 | 抗异常值 | 信息损失较多 |

#### 实验三：关键词维度的区分度

检验五大维度关键词在识别环境语句时的召回率和准确率：

| 维度 | 召回率 | 准确率 | F1 |
|------|--------|--------|-----|

| 资源利用 | ~85% | ~89% | ~0.87 |
| 绿色认证 | ~90% | ~95% | ~0.92 |
| 碳排放管理 | ~82% | ~87% | ~0.84 |

```python
@dataclass
class CompanyGreenwashResult:
    """企业漂绿测算结果"""
    # 基本信息
    company_name: str           # 企业名称
    stock_code: str             # 股票代码
    year: int                   # 年份
    industry: str               # 行业

    # 语句统计
    total_sentences: int        # 总语句数
    env_sentences: int          # 环境语句数
    substantive_count: int      # 实质性陈述数
    descriptive_count: int      # 描述性陈述数
    non_env_count: int          # 非环保语句数

    # 核心指标
    avg_env_tone: float         # 平均环境语调
    industry_median_tone: float # 行业中位数语调
    greenwash_index: float      # 漂绿指数

    # 详细结果
    sentence_results: List[SentenceLevelResult]  # 语句级详情


@dataclass
class SentenceLevelResult:
    """语句级分析结果"""
    sentence: str               # 原句文本
    category: str               # 分类（substantive/descriptive/non_env）
    confidence: float           # 分类置信度
    sentiment_score: float      # 情感得分（仅描述性语句）
    model_votes: dict           # 各模型投票详情
    fleiss_kappa: float         # 一致性系数
```

## 十二、创新点与技术亮点

### 12.1 方法论创新

1. **异构LLM集成推理框架**
   - 三个不同架构、不同训练背景的大模型形成"专家评审团"
   - 通过多数投票 + 集成平均降低个体偏差，提升结果稳定性
   - Fleiss' Kappa = 0.84，模型间高度一致

2. **二维漂绿度量体系**
   - **性质维度**：实质性 vs 描述性 — 区分"做"和"说"
   - **情感维度**：环境语调积极程度 — 量化"说得多夸张"
   - **基准对比**：行业中位数修正 — 消除行业差异

3. **全链路可解释性**
   - 每个判断都追溯到具体语句
   - 每个语句都展示三模型投票详情
   - 漂绿指数计算公式透明、可复现

### 12.2 工程实现亮点

1. **四引擎PDF解析 Fallback**
   - pdfplumber → PyPDF2 → pypdf → pdfminer 四级兜底
   - 最大限度兼容各种PDF文件

2. **Mock + 真实 双模式**
   - Mock模式：零配置、秒级响应、便于演示测试
   - 真实模式：对接API、精准分析
   - 一键切换，无缝体验

3. **森林绿主题可视化系统**
   - 完整的绿色主题配色系统（主色/辅助色/中性色）
   - 符合 WCAG AA 级无障碍对比度标准
   - 精致的卡片、按钮、图表样式
   - 响应式设计，适配多端

4. **智能企业信息识别**
   - 自动从文件名提取企业名称和年份
   - 支持「贵州茅台2023年年度报告」等多种命名格式
   - 减少用户手工输入，提升体验

5. **完善的错误处理**
   - 每一步操作都有异常捕获和处理
   - 分类清晰的错误提示（格式错误/大小超限/解析失败等）
   - 建设性的解决方案建议
